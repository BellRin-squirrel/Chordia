# 名称変更とリポジトリの過去について
このリポジトリは以前開発していたChordia_Windows（旧Chordia_Windows_Portable）とChordia_iOSの合併によるものです。
Chordia_WindowsはWindowsでもMacでも動作するように改めるため、Chordia_Desktopと名称を変更しました。（リリース名がChordia_WindowsやChordia_Windows_PortableのものはWindows版のみ対応、Chordia_Desktopの場合はWindows, Macに対応しています。）
Chordia_iOSはiOS, iPadOS, Androidのどれでも動作できるように改めるため、Chordia_Mobileと名称を変更しました。（リリース名がChordia_iOSやChordia_ipaのものはiOS, iPadOSのみ対応、Chordia_Mobileの場合はiOS, iPadOS, Androidに対応しています。）
バージョンはMobileとDesktopでのみ区別しており、iOS, iPadOS, Androidの対応についてそれぞれやWindows, Macの対応についてそれぞれ区別はしておりません。
例えば、バージョンは以下のように振られます。
Desktop版
Chordia_Windows v1.0.0
Chordia_Windows v1.0.1
Chordia_Desktop v1.1.0
Mobile版
Chordia_iOS v1.0.0
Chordia_iOS v1.0.1
Chordia_Mobile v1.1.0
Chordia_DesktopやChordia_Mobileをリリースした後にChordia_WindowsやChordia_iOSをリリースすることは原則ありません。

※2 v1.0.0-beta3_iOSは歌詞同期・歌詞表示に対応していますが、v1.0.0-beta1_Windowsやv1.0.0-beta2_Windowsから同期することはできません。<br>
※3 v1.0.0-beta1_iOS, v1.0.0-beta2_iOSは歌詞同期・歌詞表示に対応していないため、Windows版から歌詞を同期したり、歌詞を表示したりすることができません。<br>
※4 v2.0.0-beta1_iOS_ipa, v2.0.0-beta1_Windows_Portableでは同期の際のセキュリティを向上させたため、それ以前のバージョンと同期することができません。<br>
※5 v2.3.0_Windows_Portable以降ははプレイリストの楽曲順序並び替えに対応していますが、v2.1.1_iOS_ipa以前(v2.1.0(RNTP)を含む)は楽曲順序並び替えが行われたプレイリストの同期に対応していません。楽曲順序並び替えが行われたプレイリストを同期しても、タイトルを昇順で同期されます。（並び替えはiOS版には同期されません）<br>
※6 v2.1.2_iOS_ipaはバグにより楽曲を同期することができません。<br>
※7 v2.6.2_Windows_Portableは同期用のポート番号が5000固定ではなくなったため、ポート番号を設定できるv3.0.2_iOS_ipaより前のバージョンでは同期することができません。また、v3.0.2_iOS_ipaはポート番号を設定できますが、v2.6.2_Windows_Portableより前のバージョンではポート番号の表示がないため、原則同期することはできません。ただし、Windows版を複数立ち上げていない場合（または最後にWindows版を終了してからプロセスが完全に終了する程度の十分な時間を置いた場合）は、同期用ポート番号が5000に固定で割り振られ、5000はv3.0.2_iOS_ipaより前のバージョンが自動で設定していたポート番号のため、同期できる場合がございます。また、flaskサーバーのポート番号を無理やり調べることで可能になる場合もございます。
※8 v3.0.0-beta1_Windows_Portable以降はRustにより書かれており、同期速度が向上していますが、それ以前のバージョンはpythonで書かれているため、v3.0.0-beta1_Windows_Portable以降より相対的に同期速度が遅く感じる場合がございます。
※9 v2.0.0_Windows_Portable以前について改めてv4.0.0#1との同期を確認したところ、QRコードでの自動接続はできないが、IPアドレスをWindows版に表示されているものを入力し、ポート番号を5000にすると同期できることを確認しました。また、歌詞のデータも同期されることも確認済みです。
